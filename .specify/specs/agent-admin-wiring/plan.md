# Plan: Agent ↔ Admin Wiring

## Summary

Route the five existing direct-OpenAI call sites through the backend Game Master agents, starting with the Architect for quest generation. Preserve the three-tier fallback (agent → direct OpenAI → deterministic). Thread IChingContext into agent requests. Surface agent reasoning in admin UI.

## Phases

### Phase 1: Agent Client + IChingContext on Backend

- Extend backend `AgentDeps` to accept optional `IChingContext` dict (hexagramId, trigrams, tone, text, activeFace, kotterStage, nationName).
- Architect agent incorporates IChingContext into its system prompt when present (hexagram tone, Game Master face lens, campaign stage).
- Add `POST /api/agents/architect/compile` endpoint — accepts the full quest grammar compilation payload (unpacking answers, emotional signature, IChingContext, player context) and returns structured `QuestCompilation` (node text, overview, Twee source).
- Test: architect compile with and without IChingContext; verify deterministic fallback.

### Phase 2: Server Action Routing (Architect)

- Create `src/lib/agent-client.ts` — typed fetch wrapper for backend agent API (same pattern as `src/lib/agents.ts` but for server actions, not client components).
- Modify `src/actions/quest-grammar.ts`: replace `generateObjectWithCache` for quest node text, depth passage, and overview generation with calls to the backend Architect agent via `agent-client.ts`.
- Preserve cache layer: hash the same inputs, check cache first, call agent on miss, store result.
- Three-tier fallback: try agent → catch → try direct OpenAI → catch → deterministic.
- Test: quest grammar compilation works end-to-end through agent; verify cache hits skip agent.

### Phase 3: Fan Out to Other Agents

- **Storyteller bridge** (`quest-grammar.ts:1359`) → Diplomat agent: new `POST /api/agents/diplomat/bridge` endpoint.
- **Quest bridge** (`quest-grammar.ts:1481`) → Sage agent: route through `POST /api/agents/sage/consult` with quest crystallization prompt.
- **Book analysis** (`book-analyze.ts:196`) → Architect + Regent: new `POST /api/agents/architect/analyze-chunk` for batch book chunk extraction; Regent provides campaign alignment context.
- **Copy improvement** (`copy-improvement.ts:99`) → Diplomat agent: new `POST /api/agents/diplomat/refine-copy` endpoint.
- **Taxonomy extraction** (`extract-321-taxonomy.ts:63`) → Shaman agent: new `POST /api/agents/shaman/identify` endpoint for nation/archetype extraction from free text.

### Phase 4: Admin Agent Insights

- Add `AgentInsightsPanel` component to admin quest detail page (`/admin/quests/[id]`).
- Display: which agent handled generation, confidence score, discerned WAVE move, legibility note, token usage, generative deps (compostable quest links).
- Store agent metadata on quest creation: add `agentMetadata` (JSON) column to `custom_bars` — records agent name, deterministic flag, confidence, reasoning hash.
- Admin quest list: badge showing "Architect" / "Deterministic" / "Direct AI" provenance.

### Phase 5: I Ching Learning Infrastructure

- Add `HexagramEncounterLog` model (Prisma + SQLAlchemy): agentName, hexagramId, upperTrigram, lowerTrigram, isHomeTrigram, trigramPosition, interpretationSummary, archetypesInvolved, questOutcome, emotionalAlchemyTag, timestamps.
- After every agent call that includes IChingContext, log a hexagram encounter: agent name, hexagram details, whether it's the agent's home trigram, and a short interpretation generated by the agent itself (one extra inference call or inline extraction from the main response).
- Agent system prompt augmentation: before each call, query the agent's recent encounter history (last N encounters with this hexagram, or last N encounters total) and inject a "Your I Ching Journal" section into the system prompt. This gives the agent memory of its own interpretive history.
- Periodic synthesis: a scheduled or admin-triggered job that asks each agent to review its encounter log and produce a short interpretive profile update ("What have I learned about hexagram X?"). Store as `AgentInterpretiveProfile`.
- Home trigram awareness: when an agent encounters a hexagram containing its own trigram, flag it. The agent should know when it's working in resonance vs. creative tension.

### Phase 6: Sage Orchestration for Complex Quests

- Wire the Sage as the entry point for quest grammar compilation when the quest involves multiple WAVE stages or cross-domain alignment.
- Sage consults Architect (structure), Shaman (emotional tone), Challenger (move energy check), then synthesizes.
- Sage draws on its own Mountain-trigram interpretive profile to determine which agents to consult for a given hexagram.
- Admin can trigger "Sage Review" on any quest from the quest editor.

### Phase 7: Admin I Ching Learning Dashboard

- Encounter log viewer per agent: filterable by hexagram, trigram, date, home-trigram encounters.
- Pattern visualization: which hexagrams each agent encounters most, quality distribution by hexagram.
- Interpretive profile editor: admin can view, curate, or reset an agent's emerging profile.
- Emergence timeline: track how an agent's interpretation of a specific hexagram changes over encounters.

## Dependencies

- Backend must be running alongside Next.js (Makefile `up` target handles this)
- `NEXT_PUBLIC_BACKEND_URL` env var (defaults to `http://localhost:8000`)
- Agent Mind Model spec (completed — agents exist and work)
- Quest Grammar Compiler spec (completed — compilation pipeline exists)

## Generative Dependencies

Completing Phase 2 (Architect routing) eliminates the need for:
- Maintaining two separate prompt engineering approaches (agent system prompts vs quest-grammar prompts)
- Duplicating DB context loading (agents already have tools for this)
- Manual IChingContext prompt injection (agents get it structurally)

Completing Phase 4 (Admin Insights) eliminates the need for:
- Separate "AI audit" logging — the agent response envelope is the audit trail

Completing Phase 5 (I Ching Learning) eliminates the need for:
- Hand-coding per-agent I Ching interpretation rules — agents build their own over time
- Separate "narrative tuning" for different hexagram contexts — the encounter log is the tuning data
- Static prompt engineering for hexagram tone — agents learn which tones work through their own encounters
