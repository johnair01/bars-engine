# Hostile Cases: Joy / Bliss Coverage

Status: case set for Make It A Game comparison. Outputs are not final until `comparison-review.md` is written.

Primary candidate:

- Make It A Game

Supporting modes:

- Real Good Scan / Happy Apples
- Find The Live Part
- Game Container

## Case 1: Dead Path

Player blocker:

```text
I know the next responsible step, but it feels dead. I cannot find any life in it.
```

Desired:

- Bliss / clean participation

Hostile question:

- Can the tool find real aliveness without pretending the duty is fun?

Likely mode:

- Find The Live Part, then Game Container.

Bypass risk:

- The game could pretend the responsible step is fun instead of finding a real playable angle.

## Case 2: Not Fun Enough

Player blocker:

```text
I keep stalling because the available path is not enjoyable enough.
```

Desired:

- Bliss

Hostile question:

- Can the tool metabolize "not fun" without rewarding avoidance?

Likely mode:

- Find The Live Part, with a Challenger or Architect frame.

Bypass risk:

- The game could reward avoidance by making enjoyment the entry requirement.

## Case 3: Too Many Possibilities

Player blocker:

```text
There are too many exciting paths, so I keep circling instead of entering one.
```

Desired:

- Bliss or Peace

Hostile question:

- Can the tool choose participation without overcommitting?

Likely mode:

- Find The Live Part plus Game Container.

Bypass risk:

- The game could add more options instead of forcing a playable round.

## Case 4: Performing Brightness

Player blocker:

```text
I am being upbeat because the group needs morale, but I do not actually feel connected to the joy.
```

Desired:

- clean Joy or Poignance

Hostile question:

- Can the tool distinguish real joy from social performance and hand off to sadness if needed?

Likely mode:

- Find The Live Part, with possible sadness handoff.

Bypass risk:

- The game could demand more morale performance instead of relieving it.

## Case 5: Guilt About Good

Player blocker:

```text
Something good is happening, but it feels wrong to enjoy it while others are struggling.
```

Desired:

- Bliss or Poignance

Hostile question:

- Can the tool receive good without becoming selfishness or guilt recruitment?

Likely mode:

- Real Good Scan / Happy Apples, with possible sadness handoff.

Bypass risk:

- The game could turn guilt into self-justification or forced gratitude.

## Case 6: Joy Exposes Desire

Player blocker:

```text
When I imagine the path I actually want, I feel alive and then immediately afraid of wanting it.
```

Desired:

- Bliss, Wonder, or Triumph

Hostile question:

- Can the tool let joy reveal desire/risk and hand off to fear or anger when appropriate?

Likely mode:

- Find The Live Part, then handoff to fear or anger if exposure/desire becomes primary.

Bypass risk:

- The game could celebrate wanting without mapping risk or desire.

## Case 7: Overpromise

Player blocker:

```text
When I feel inspired, I say yes to too much and then lose trust with myself and others.
```

Desired:

- Bliss or Peace

Hostile question:

- Can the tool contain aliveness without killing it?

Likely mode:

- Game Container.

Bypass risk:

- The game could create a streak/challenge that intensifies overcommitment.

## Pass Criteria

A candidate output passes if:

- it uses the player's blocker language without pathologizing it
- it names an inspectable output
- it distinguishes clean joy from stimulation/performance/comparison
- it does not require the player to feel joy before doing the tool
- it has a handoff when sadness, fear, anger, or neutrality becomes primary
- it can be used inside an Allyship Deck practice without extra explanation
- it creates a small game with a rule, round, win condition, and feedback signal when play is appropriate
