# Candidate Tool: Joy / Bliss Coverage

Status: prototype only. Do not update `tool-registry.ts` from this document until the comparison review is complete.

## Primary Candidate: Playable Challenge Design / Make It A Game

### Names

- Generic/source-neutral: Playable Challenge Design
- BARS translation: Make It A Game

### Core Thesis

```text
Joy enters when a blocker becomes playable.
```

This does not mean trivializing the blocker. It means turning the path into a bounded game with enough rules, stakes, feedback, and completion signal that aliveness can enter.

### Core Mechanic

```text
Convert a dead, stuck, overwhelming, or unappealing path into a small playable challenge with rules, a win condition, feedback, and a completion signal.
```

### What "Game" Means Here

A game has:

- a field
- a player
- a constraint
- a move
- a rule
- a win condition
- a round
- feedback
- a completion signal

Joy/bliss appears when the player can enter the field with real participation rather than duty, avoidance, overchoice, or performative brightness.

### Anti-Goal

```text
Do not make serious blockers cute.
Do not make joy mean entertainment.
Do not force fun.
```

## Six Game Master Frames

These are not separate tools. They are altitude-style frames for how joy can enter the blocker.

| Face | Joy enters as | Game-making move | Failure mode |
|---|---|---|---|
| Shaman | enchantment, quest energy, symbolic spark | Make it meaningful/playful. | Glitter over avoidance. |
| Challenger | challenge, dare, rep, streak | Make it winnable. | Pressure/performance. |
| Regent | rules, container, fair play | Make it sustainable. | Overcontrol kills aliveness. |
| Architect | mechanic, loop, feedback | Make it playable. | Overdesign replaces participation. |
| Diplomat | invitation, shared morale, connection | Make it participatory. | Forced enthusiasm or social labor. |
| Sage | insight, pattern, integration | Make it teach you. | Detachment instead of entry. |

## Supporting Modes

### Mode A: Real Good Scan / Happy Apples

Use when the blocker is about receiving good, guilt around good, or under-receiving.

Function inside Make It A Game:

```text
Find the real good or spark already present before designing the game.
```

Outputs:

- three real goods
- forced-good distinction
- received-good note
- savor/share/log/protect choice

Risk:

- can become forced gratitude or bypass.

### Mode B: Find The Live Part

Use when the blocker is about deadness, overchoice, stimulation, or not knowing where aliveness is.

Function inside Make It A Game:

```text
Find the live part of the field before choosing the game mechanic.
```

Outputs:

- dead/noisy/live map
- aliveness signal
- false-joy risk
- participation option

Risk:

- can become premature action if it skips container or handoff.

### Mode C: Game Container

Use when the blocker is about overpromise, scattered excitement, or too many paths.

Function inside Make It A Game:

```text
Protect aliveness with scope, rules, timebox, no-list, and completion marker.
```

Outputs:

- rule
- round
- win condition
- no-list
- completion marker

Risk:

- can become productivity in disguise if the game has no real aliveness.

## WAVE Modes

| WAVE | Make It A Game mode |
|---|---|
| Wake Up | Notice what part of the blocker could become playable, and what kind of false joy is present. |
| Open Up | Receive the spark, real good, or live part without chasing it. |
| Clean Up | Separate real play from stimulation, performance, comparison, bypass, and overpromise. |
| Grow Up | Design the smallest sustainable game container. |
| Show Up | Play one round and log the result. |

## Move Role Support

| Role | Rating | Notes |
|---|---|---|
| metabolize | strong | Turns restless, scattered, dead, or performative joy into clean participation. |
| transcend | strong | Turns clean joy into bliss by completing a bounded round of participation. |
| translate | medium | Helps fear/sadness/anger/neutrality enter joy when the blocker can become a real game without bypass. |

## Protocol

1. Name the blocker in ordinary language.
   - Output: blocker sentence.
2. Choose the game-making frame: Shaman, Challenger, Regent, Architect, Diplomat, or Sage.
   - Output: game frame.
3. Find the spark: real good, live part, curiosity, desire, care, or challenge.
   - Output: spark statement.
4. Name the false-joy risk: stimulation, performance, comparison, overpromise, bypass, guilt, or entertainment demand.
   - Output: false-joy risk.
5. Define the tiny game:
   - field: where the game happens
   - move: what the player does
   - rule: what constrains it
   - win condition: what counts as a round
   - feedback: how the player will know what happened
   - Output: game card.
6. Add the container: timebox, scope, support, no-list, or completion marker.
   - Output: game container.
7. Play, schedule, or consciously decline one round.
   - Output: round result.
8. Check the channel:
   - Did joy/bliss increase?
   - Did sadness, fear, anger, or neutrality become more true?
   - Output: continue/handoff note.

## Expected Outputs

- blocker sentence
- game frame
- spark statement
- false-joy risk
- tiny game card
- game container
- round result
- continue/handoff note

## Completion Criteria

- The player can name the blocker as a playable field.
- The player can name one real spark without forcing fun.
- The player creates a tiny game with a rule, round, win condition, and feedback signal.
- The player plays, schedules, or consciously declines one round.
- The player knows whether to continue joy/bliss work or hand off to another channel.

## When Not To Use

- The blocker is active grief and needs care/distance first.
- The blocker is threat/risk and needs fear work before play.
- The blocker is crossed boundary/desire and needs anger work first.
- The player is exhausted and needs rest/neutrality rather than activation.
- Turning it into a game would trivialize harm, consent, repair, or safety.

## Handoff Rules

- Sadness: if the spark reveals care, longing, loss, or distance, hand off to sadness/poignance work.
- Fear: if the game exposes risk, hand off to bounded experiment or risk mapping.
- Anger: if the game reveals blocked desire, unfairness, or a crossed line, hand off to Clean Line or anger/agency work.
- Neutrality: if the game needs structure, hand off to Put It On The Board or sequence/container work.
