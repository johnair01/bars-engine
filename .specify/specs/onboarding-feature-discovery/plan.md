# Plan: Onboarding Feature Discovery and Send Experience

## Overview

Clean up the onboarding send experience so players know how to make BARs, create quests, use EFA, and donate. Content exists as quests (actionable) and documentation (reference). Goal: excite players to use the app and move the needle.

## Phases

### Phase 1: Documentation

- [ ] BARs: Add/expand Player Handbook or docs section—what a BAR is, how to create, BAR-to-quest, BAR-on-quest
- [ ] Quests: Add quest creation guide; subquests; campaign quest flow
- [ ] EFA: Add EFA guide (when to use, what it does, vibeulon moves)
- [ ] Donation: Document donation flow; ensure clear path from dashboard/campaign

### Phase 2: Quest Content

- [ ] BAR discovery quest: What a BAR is, how to create one, how it connects to gameplay
- [ ] Quest discovery quest: How to make a quest, add subquests to campaign quests
- [ ] EFA discovery quest: How to use EFA, when, what it does
- [ ] Donation prompt: Surface donation as a way to support (optional quest or banner)

### Phase 3: Send Flow Integration

- [ ] Integrate discovery quests into orientation or post-onboarding flow
- [ ] Ensure Game Map, Library, EFA, Gameboard are discoverable from send
- [ ] Donation link visible from dashboard, campaign, event page
- [ ] Cross-link quests ↔ docs

### Phase 4: Tone and Excitement

- [ ] Review copy for inviting, exciting tone (not instructional)
- [ ] Frame features in terms of impact: BARs fuel quests; quests advance campaign; EFA unblocks; donations support

## Dependencies

- Game Map and Lobby Navigation (FP)
- Dashboard Orientation Flow
- K-Space Librarian (if applicable)
- Event/Donation flow
