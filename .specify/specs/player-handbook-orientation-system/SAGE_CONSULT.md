# Sage Consult — Ontology Dislocation

**Date:** March 2026  
**Consulted agents:** Shaman, Diplomat, Architect  
**Discerned move:** Wake Up  
**Hexagram:** 39 — *Attunement before acting. Align timing and energy before proceeding.*

## Question posed

> We have an ontological blocker in BARS Engine. Three foundational books have been added to the library (Antifragile, Wealth of Networks, Complete Focusing Instructions). The blocker shows up as four consistent pain points: navigation is confusing and unintuitive; orientation to the game is unclear; there is no player's handbook; there is no pathway from admin library knowledge to players. These are not four separate UI bugs — they are one underlying problem. How do we name it precisely, and what is the minimal intervention?

## Synthesis

> The core underlying problem is best named **'Ontology Dislocation'**. This term captures the disconnect between the rich knowledge architecture and the player experience. The solution is to create an **Ontology Bridge Layer** that serves not only as connective tissue but as a player guide that unfolds dynamically as players engage with the game mechanics.
>
> A minimal effective intervention:
> 1. **Navigation Harmonization** — align concepts and functions with intuitive player interactions.
> 2. **Interactive Player Guide** — an unfolding handbook that grows with each player action.
> 3. **Smart Alerts and Feedback** — prompts that use player actions to navigate gamespace.
> 4. **Bridge Quests and Challenges** — quests that engage players directly with systemic concepts.
>
> Hexagram 39 suggests deliberate pacing — develop with careful sequencing so players are universally up-leveled rather than overwhelmed.

## Annotation (spec author)

The four pain points are **one missing layer** sitting between the game's rich backend ontology (moves, quests, books, nations, archetypes, 321, charge) and the player's lived experience. Everything the system knows about how to help a player is locked in admin, wiki pages, and code — none of it woven together as a **player's living journey**.

The Ontology Bridge Layer is not a feature. It is a **connective tissue** spec that orchestrates:
- [player-main-tabs-move-oriented-ia](../player-main-tabs-move-oriented-ia/spec.md) — IA analysis
- [library-praxis-three-pillars](../library-praxis-three-pillars/spec.md) — foundational readings as pillars
- The handbook, the move compass, the library pipeline, and felt-sense scaffolding

Hexagram 39 is a constraint: **do not rebuild the nav in one sprint.** Each phase must be independently valuable.
