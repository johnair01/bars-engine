# Primitive Translation Proofs v0.1

## Purpose

This artifact proves the primitive-first architecture:

```text
primitive first, translation second, card third
```

The goal is not to finalize deck copy. The goal is to prove that one primitive can generate different playable Show Up moves through orientation, subject, superpower, domain, and blocker.

## Proof Grammar

Each proof uses:

| Field | Purpose |
|---|---|
| Primitive | The emotional-alchemy mechanic. |
| State Vector | The charge movement the primitive serves. |
| Orientation | Internal or external Show Up. |
| Subject | Self, other, or collective. |
| Superpower | Style of expression. |
| Domain | Field/output type. |
| Blocker | Friction that shapes scale and completion. |
| Generated Move | The contextual Show Up move. |
| Done When | Completion standard. |
| Afterward Notice | Reflection prompt for drift/learning after the move. |

## Proof 1: MP02 Internal

| Field | Value |
|---|---|
| Existing prototype | MP02 Resource Ask Constraint |
| Primitive | `bound_the_ask` |
| State Vector | `fear:dissatisfied -> fear:neutral`; uncertainty becomes clean discernment. |
| Orientation | Internal |
| Subject | Self |
| Superpower | Strategist |
| Domain | Gather Resources |
| Blocker | The ask is too vague. |

### Generated Move

Create a personal ask constraint:

```text
When I need support for ___,
I ask ___ for ___
by ___
so that ___ can move.
```

Fill each blank once. Keep the ask small enough that you can imagine making it without needing to become a different person first.

### Done When

The ask constraint is written and tied to a use trigger:

```text
If I am still blocked by ___ at ___, I use this ask.
```

### Afterward Notice

Did the constraint make asking more possible, or did it become another way to delay asking?

### What This Proves

The primitive can complete internally. No ask has been sent, but capacity has been enacted because the player now has a reusable ask structure and a trigger.

## Proof 2: MP02 External

| Field | Value |
|---|---|
| Existing prototype | MP02 Resource Ask Constraint |
| Primitive | `bound_the_ask` |
| State Vector | `fear:dissatisfied -> fear:neutral`; uncertainty becomes clean discernment. |
| Orientation | External |
| Subject | Other or collective |
| Superpower | Strategist |
| Domain | Gather Resources |
| Blocker | The ask is too vague. |

### Generated Move

Send or schedule one bounded ask:

```text
I am asking for ___
because ___
by ___.
The next step would be ___.
```

Send it to one named person or group. If sending now would be unsafe or unwise, schedule the send and name the condition that would make it clean.

### Done When

The bounded ask is sent, or scheduled with a named recipient and send condition.

### Afterward Notice

What changed when the ask left your head and entered the field?

### What This Proves

Same primitive, same superpower, same domain, different orientation. External completion requires field contact or a concrete scheduled contact.

## Proof 3: MP08 Internal

| Field | Value |
|---|---|
| Existing prototype | MP08 Clean Exit Map |
| Primitive | `clean_exit` |
| State Vector | `fear:dissatisfied -> fear:neutral -> wonder`; threat becomes option and agency. |
| Orientation | Internal |
| Subject | Self |
| Superpower | Escape Artist |
| Domain | Direct Action |
| Blocker | Leaving may disappoint people. |

### Generated Move

Create a clean inner exit:

```text
The cage I am naming is ___.
The obligation that is real is ___.
The obligation that is guilt or theater is ___.
The clean exit I allow myself is ___.
What I carry forward is ___.
```

### Done When

You have a written refusal rule or inner exit commitment that can be used when the pressure returns.

Example:

```text
When I feel guilt pressure to stay in ___, I pause and choose from ___ instead of automatically complying.
```

### Afterward Notice

Does the exit create more agency, or does it hide accountability you still need to carry?

### What This Proves

An exit can be an inner Show Up move. The player has not left externally, but fear has become usable agency through an enacted refusal rule.

Create a non-glue boundary:

```text
The invisible labor I have been holding is ___.
```


### What This Proves

`create_handoff` can complete internally as a self-allyship move. The handoff begins by withdrawing the self from being the hidden infrastructure.

## Proof 6: MP19 External

| Field | Value |
|---|---|
| Existing prototype | MP19 Exit The Glue Role |
| Primitive | `create_handoff` |
| State Vector | `sadness:dissatisfied + anger:dissatisfied -> neutrality:neutral`; overheld care becomes shared structure. |
| Orientation | External |
| Subject | Collective |
| Superpower | Connector |
| Domain | Skillful Organizing |
| Blocker | If I stop holding it, things may fall apart. |

### Generated Move

Create one handoff agreement:

```text
I have been holding ___ informally.
For this to keep working, it needs ___.
Can we agree that ___ owns ___ by ___?
```

The Connector expression is not "do the handoff yourself." It is to create enough trust and clarity that the handoff can survive without you as glue.

### Done When

One invisible labor item becomes a named role, rotation, owner, checkpoint, or agreement.

### Afterward Notice

Did the handoff create shared structure, or did you remain the bridge everyone still depends on?

### What This Proves

Same primitive, same superpower/domain, different orientation. External completion requires a shared agreement or visible structural change.

## Proof 7: MP05 External

| Field | Value |
|---|---|
| Existing prototype | MP05 Interrupt The False Calm |
| Primitive | `interrupt_pattern` |
| State Vector | `anger:dissatisfied -> anger:neutral`; frustration becomes clean boundary/desire. |
| Orientation | External |
| Subject | Collective |
| Superpower | Disruptor |
| Domain | Raise Awareness |
| Blocker | Everyone is staying polite. |

### Generated Move

Ask one pattern-interrupting question:

```text
What are we not saying because politeness is safer than truth?
```

Before asking, name the pattern you are interrupting, not the person you are blaming.

Use a question that makes reality visible without turning the room into a trial.

### Done When

The question is asked in the relevant room/thread, or sent to the accountable group.

### Afterward Notice

Did the question reveal the pattern, or did it make you the spectacle?

### What This Proves

`interrupt_pattern` can translate through Disruptor + Raise Awareness as visibility rather than force. Same primitive can serve a truth-making domain.

## Proof 8: MP12 External

| Field | Value |
|---|---|
| Existing prototype | MP12 Pattern Interruption |
| Primitive | `interrupt_pattern` |
| State Vector | `anger:neutral -> anger:satisfied`; clean boundary/desire becomes triumph through precise force. |
| Orientation | External |
| Subject | Collective or other |
| Superpower | Disruptor |
| Domain | Direct Action |
| Blocker | Politeness protects harm. |

### Generated Move

Stop the next repetition:

```text
The repeating harm is ___.
The next likely repetition is ___.
The line I will hold is ___.
The person/value protected is ___.
```

Then make the interruption at the next repetition: state the line, refuse the pattern, or redirect the action.

### Done When

The next repetition is interrupted, or a boundary is placed before it can repeat.

### Afterward Notice

Did the interruption protect the named person/value, or did it escalate beyond the needed line?

### What This Proves

`interrupt_pattern` can translate through Disruptor + Direct Action as precise intervention, not merely awareness. Same primitive, different domain output.

## What This Proves

The architecture works.

One primitive can generate multiple valid moves without authoring each combination from scratch.

The proof set also shows that:

- orientation changes completion;
- domain changes output type;
- superpower changes style;
- blocker changes scale and safety;
- reflection detects drift after the move.

## Remaining Design Questions

- Should internal translations always produce written artifacts, or can embodied commitments count if logged afterward?
- Should "prepared with safety reason" count as external completion, or should it be a separate blocked-state outcome?
- Should the same primitive expose multiple difficulty levels, such as tiny rep, standard move, and strong move?
- Should reflection prompts be generated by primitive, superpower, or blocker?
