# Hostile Review v2: Inner And Outer Show Up

## Corrected Premise

The first hostile review found real problems, but it used the wrong primary test.

Wrong test:

```text
Show Up must touch the external world.
```

Correct test:

```text
Show Up must enact transformed capacity.
```

That enactment can be internal or external:

- **Internal Show Up:** the player creates an inner artifact or commitment they can actually live from: permission, inner boundary, values bridge, self-trust note, clean exit, personal plan, parts agreement, refusal rule, or embodied practice.
- **External Show Up:** the player creates contact in a relationship, campaign, or field: ask, message, handoff, interruption, agreement, resource movement, repair, public truth, or witnessed action.

So this review does not fail a move for being internal. It fails a move when it is only analysis, mood management, planning, or naming without enacted capacity.

## What The Review Now Says The Set Is Missing

The missing fields are not cosmetic. They are what make the cards testable.

| Missing Field | Why It Matters |
|---|---|
| Orientation | Determines whether the completion is inner artifact or outer contact. |
| Subject | Distinguishes self-allyship, allyship for another, and collective/campaign work. |
| State Vector Notation | Keeps the move connected to the 15-state graph rather than vague feeling words. |
| Vector Mechanic | Explains what the charge can now do. |
| Superpower Expression | Explains what the superpower contributes that the domain alone would not. |
| Domain Output | Names the artifact/action type the domain requires. |
| Enacted Artifact / Act | Prevents reflection from masquerading as Show Up. |
| Completion Standard | Makes "done" observable for internal and external moves. |
| Shadow Fail Condition | Converts drift questions into playable constraints. |

## Revised v0.2 Card Grammar

| Field | Description |
|---|---|
| Prototype ID | Stable ID, e.g. MP08A. |
| Scenario ID | Golden scenario fixture. |
| State Vector | Canonical notation plus plain language. |
| Vector Mechanic | What changes in the charge after the move. |
| Superpower | Connector, Strategist, Disruptor, Escape Artist, Catalyst/Coach, Alchemist, Storyteller. |
| Superpower Expression | How that superpower uniquely expresses the vector. |
| Domain | Gather Resources, Raise Awareness, Direct Action, Skillful Organizing. |
| Domain Output | Ask, signal, boundary, route, agreement, resource, story, repair, etc. |
| Orientation | Internal or external. |
| Subject | Self, other, or collective. |
| Card Context | Exact deck card ID or declared card family. |
| Blocker | The obstruction that changes the move. |
| Pre-Move | Any required Wake/Open/Clean/Grow preparation. |
| Show Up Artifact / Act | The enacted inner artifact/commitment or outer contact/action. |
| Completion Standard | Observable done condition. |
| Shadow Fail Condition | When the move should not be used or how it drifts. |

## Bottom Line

The second review keeps the good pressure from the first review but removes the false equation:

```text
external = Show Up
internal = not Show Up
```

The new equation is:

```text
Show Up = enacted capacity.
```

Internal Show Up must produce something the player can live from. External Show Up must produce contact in the field. Both can be real. Both can also be fake if they bypass the actual emotional vector.
