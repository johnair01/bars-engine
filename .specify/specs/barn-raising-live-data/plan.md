# Plan: Barn Raising — Live Data

> Implement per [.specify/specs/barn-raising-live-data/spec.md](./spec.md). **API-first**:
> land the schema + `getBarnSnapshot` contract before wiring the UI read.

## Architecture strategy

Thin, additive layer over existing milestone/donation infra. One nullable column, one read
action, one seed, and a small write-path extension. The presentational `BarnRaisingBar`
already accepts `state` — we only feed it real data.

## Sequencing

1. **Schema + migration** (FR1) — add `wallKey`; `migrate dev`; commit migration; `db:record-schema-hash`.
2. **Seed** (FR2) — `seed-barn-raising.ts`; verify rows created idempotently.
3. **Read contract** (FR3) — `getBarnSnapshot`; unit-cover the cents conversion + hands/beams aggregation.
4. **UI read** (FR4) — wire `/event/barn` + `/pricing` teaser; preserve preview + empty fallback.
5. **Write path** (FR5) — thread query tags through donate → `dswMeta` + `recordContribution`.
6. **Keep-building** (FR6) — wall-complete redirect via guidance pattern.
7. **Verification quest** — Twine passages + seed; `seed:cert:barn-raising-live`.
8. **Fail-fix** — `npm run build`, `npm run check`.

## Risks
- **Cents vs float**: `CampaignMilestone.currentValue` is a `Float` (dollars in some flows, cents in others) — confirm the unit the existing donate path writes and convert consistently in `getBarnSnapshot`. Add a test.
- **DB-down public render**: `/event/barn` is public; must not 500 on preview deploys — wrap reads.
