# Polarity Design Session — MTGOA + Friendcraft

**Cast statement:** Before writing Friendcraft book 64 cards or revising MTGOA hexagram integration, lock **governing polarities first** (Phase A), then **structural axes** for each exploration deck (Phase B). Scene Atlas polarity resolution (pair1 × pair2 → four corners) is the **application-layer** pattern; I Ching 8×8 is the **exploration-layer** pattern — confirmed for both allyship and friendcraft.

**Phase order (mandatory):**
- **Phase A** — Governing polarities interview (`GOVERNING-POLARITIES-INTERVIEW-PROTOCOL.md`)
- **Phase B** — Structural matrix: MTGOA domain integration + Friendcraft 8+8 trigram identities (`I-CHING-EXPLORATION-STRUCTURE.md`)

**Why one session, two lines:** Scene Atlas proved the **2×2 engine pattern**. MTGOA has a **64 matrix** (Gate × Chapter). Friendcraft adopts the **same 8×8 exploration structure** with friendship-native content — not a third topology.

## The shared pattern (Scene Atlas)

```
pair1:  poleA (−)  ↔  poleB (+)
pair2:  poleC (−)  ↔  poleD (+)
        ─────────────────────────
        4 quadrants  →  4 suit keys  →  52 ranks on application layer
```

**Resolution order (engine):** adventure/orientation JSON → nation + playbook derivation → defaults.

**Design split:**
- **WCGS** = developmental function of the four suits (fixed per craft line)
- **Polarity** = how quadrants are *labeled* for this player (derived or chosen)

## Track A — MTGOA (Mastering Allyship)

### What already exists

| Layer | Structure | Status |
|-------|-----------|--------|
| **Exploration 64** | Gate (8 voices / EH trigrams) × Chapter (8 faces / LH trigrams) | 64 cards drafted; earn via BAR gate capture |
| **Application 52** | WCGS suits × 13 ranks | Workshop / practitioner decks; Casey named-person variant |
| **Allyship domains** | 4 Show Up action types (resources, organizing, direct action, awareness) | Spec'd; **not integrated** into card format |
| **Scene Atlas polarity** | pair1 × pair2 → 4 grid suits | **Not wired** to MTGOA decks |

### Session questions (MTGOA)

1. **Is Gate × Chapter already the exploration polarity?**  
   - Interior voice (gate) × exterior face (chapter) = 64 hexagram *types*  
   - If yes: polarity session = **validate mappings**, not invent new axes  
   - If no: what's missing for *player-resolved* labels?

2. **Where do allyship domains sit?**  
   - Option A: Fifth dimension (chosen at draw time)  
   - Option B: Second pair on **application** 52 only (domain × WCGS)  
   - Option C: Filter on exploration 64 (same hexagram, domain-colored task)  
   - Option D: Appendix routing only (current card footers)

3. **Does MTGOA application 52 need Scene Atlas-style pair resolution?**  
   - Practitioner deck suits = WCGS (fixed)  
   - Should suit *labels* personalize via nation/playbook/orientation like Scene Atlas?

4. **Earn path + polarity:** When BAR capture unlocks hexagram card, does unlock key include domain or polarity state?

### MTGOA deliverables (session output)

- [ ] Decision: Gate × Chapter **is / is not** the canonical exploration polarity (document rationale)
- [ ] Decision: Allyship domains placement (A/B/C/D above)
- [ ] If application polarity yes: pair1/pair2 source for allyship (nation×playbook or book-specific)
- [ ] Update list: which hexagram cards need domain pass (per ALLYSHIP_DOMAINS_SPEC open items)
