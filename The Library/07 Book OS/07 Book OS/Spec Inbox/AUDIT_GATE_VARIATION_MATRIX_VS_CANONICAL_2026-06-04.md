# Gap Audit — Gate Variation Matrix vs GATE_VOICES_CANONICAL

## Purpose

Before promoting the Gate Variation Matrix into `GATE_VOICES_CANONICAL.md`, identify:

1. What is **strong in the canonical file** that must be preserved
2. What is **missing from the canonical file** that the matrix adds
3. What is **strong in the matrix** that is genuinely new and must be merged
4. What is **redundant or weaker** in the matrix and can be dropped
5. What **structural form** the promoted file should take

| File | Words | Sections | Voice-level / Chapter-level |
|------|-------|----------|------------------------------|

## What is Missing from GATE_VOICES_CANONICAL (matrix adds)

These are the gaps the matrix fills:

- **No chapter-altitude variation anywhere in the canonical file.** The canonical gates describe the voices in the abstract. They never say "what is the Protector actually protecting **in the Shaman chapter**" vs. "in the Regent chapter." This is the book's primary structural need and the matrix's entire reason for existing.
- **No per-chapter anchor language.** Each chapter teaches a different role-pattern (Shaman=feeling, Regent=inheritance, Architect=design, Diplomat=terms, Sage=altitude, Player=play) but the canonical gates don't translate their abstract questions into chapter-specific questions.
- **No "spot-edit sheet"** — no fast-pass question shapes a writer can drop into a chapter and immediately see whether the gate is doing the chapter's work. The canonical file is conceptual; the matrix is operational.
- **No rule of thumb for each gate's role** in one paragraph (e.g., "Protector is about what is being protected from loss, exposure, or premature movement"). The canonical file has this implicitly but never distills it into a single reference line per gate.

## What is Strong in the Matrix (genuinely new, must merge)

- **The altitude assignments per chapter** (Shaman=feeling, Challenger=willing/pushing, Regent=responsibility/inheritance, Architect=design/standards/system, Diplomat=terms/staying/relationship, Sage=altitude/clarity/commitment, Player=building/play/handoff). This is the actual book structure that was missing from the canonical file.
- **The "Rule of Thumb" block at the top** — a one-paragraph description of what each gate is doing, written as a writer's reference. This is operationally useful in a way the canonical file's "Core Function" is not.
- **The 8-gate × 7-chapter anchor lists** — even in the abstract form (not the spot-edit form), this is the bridge from voice to chapter.
- **The closing principle**: "The repetition is in the walk. The variation is in the question." This is a strong, succinct line the canonical file can adopt.

## What is Redundant or Weaker in the Matrix (can drop)

- The matrix's spot-edit sheet restates the chapter anchors in question form. The question-form is useful for editing, but the canonical file is a reference, not an editing pass. **Recommendation:** keep both forms in the canonical file but mark the spot-edit sheet as an optional operational section, not the primary content.
- The matrix has no provenance, no lineage, no AQAL pairing, no trigrams, no developmental arcs. **None of this is redundant** — the canonical file owns all of it.

## Promotion Strategy

The canonical file should adopt a **layered structure**:

1. **Front matter (preserved from canonical):** Intellectual lineage, hierarchy, WAVE/gate pairing table with trigrams, source attribution.
2. **Per-gate section (new structure):** For each of the 8 gates, the canonical deep content followed by the new material:
   - Rule of Thumb (one paragraph — from matrix)
   - Existing canonical content (Core Function, How It Shows Up, Shadow, Gift, Repurpose Move, Developmental Arc, etc.)
   - **New: Chapter-Altitude Variation** (the matrix's per-chapter anchor list)
   - **New: Spot-Edit Question** (the matrix's "what to ask at this gate in this chapter" — single sentence per chapter, useful as a writer's quick reference)
3. **Cross-Gate Structural Map (preserved):** Hierarchical dependencies, pair codependencies, Wake Up → Show Up sequence.
4. **New closing section:** "The repetition is in the walk. The variation is in the question." (from matrix)

This layered structure gives the canonical file:

- The provenance and deep framework it already had
- The chapter-by-chapter variation the matrix introduces
- A spot-edit reference writers can use without re-reading the whole document

## Risks

- **Length.** The canonical file is already 32KB / 4,988 words. Adding the matrix per-chapter variation (8 gates × 7 chapters of anchor lines) adds approximately 1,500–2,000 words. The promoted file will be ~7,000 words. This is still a reference document, not a manuscript. Acceptable.
- **Drift between canonical and matrix wording.** The matrix's "Rule of Thumb" and the canonical file's "Core Function" describe the same gate in different language. The promotion must pick one register per section — the canonical voice (deeper, lineage-aware) for the deep content, the matrix's compressed writer-reference voice for the Rule of Thumb.
- **Provenance drift.** The matrix was authored 2026-06-04. The canonical file's "last updated" frontmatter should reflect the merge date and reference the matrix as a related document.
