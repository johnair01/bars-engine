# Sourcebook Integration Notes - v0.2

Purpose: record what the v0.2 MTGOA RPG manual borrows structurally from each source text.

## Hearts Blazing

Useful patterns:

- Ensemble roles instead of single-hero design.
- Shared setting generation before play.
- Caretakers for different dimensions of the world.
- Episode deck as the pacing engine.
- Pilot / Midseason / Finale arc.
- Launch / Bridge / Wrap scene structure.
- Commercial Break as a scene transition and safety valve.
- Hidden motives and keywords as dramatic pull.
- Frailty cards as character-depth bids.
- GMless or guide-light play.

MTGOA integration:

- Ensemble Roles become Spark, Anchor, Builder, Scout, Witness.
- Motives become allyship temptations: Recognition, Control, Rescue, Justice, Belonging, Liberation.
- Keywords become evidence of what the character is drawn toward, not a moral score.
- Episode Bidding gives the table a gameable way to decide pressure and narration rights.

## Dungeon World

Useful patterns:

- Fiction-first move triggers.
- 10+ / 7-9 / 6- outcome ladder.
- GM agenda, principles, and moves.
- Soft and hard consequences.
- Fronts as organized offscreen pressure.
- End-of-session questions.
- Custom moves as modular mechanics.
- "Ask questions and use the answers" as core procedure.

MTGOA integration:

- Basic Moves become Wake Up, Clean Up, Grow Up, Show Up.
- Guide Agenda focuses on living city, concrete allyship, impact, repair, and safety.
- Guide Moves convert abstract allyship failure into fictional consequences.
- Fronts become pressures like institutional defensiveness, optics campaigns, and resource depletion.
- End-of-session questions generate Legacy.

## Foundations

Useful patterns:

- Map as changing artifact rather than pretty illustration.
- Chronicle as historical memory.
- Card prompts as worldbuilding engine.
- Suit sequencing and optional wild cards.
- Large timescale and systems-level view.
- Prompts that alter existing geography, culture, history, and magic.

MTGOA integration:

- The Living City map changes after every Episode.
- Chronicle records trust shifts, resource shifts, harms, repairs, and unanswered questions.
- Districts map to allyship domains.
- Static cards function like destabilizing optional prompts.
- Campaign play tracks social/systemic change, not only character action.

## Kids on Bikes

Useful patterns:

- Boundary setting before character creation.
- Collaborative town creation.
- Rumors as playable hooks.
- Relationship questions.
- Failure creates Adversity Tokens.
- Shared special character / focal point.
- GM as facilitator who reads the room.
- Multiple-session town changes.

MTGOA integration:

- Boundary Stack becomes core safety procedure.
- Players create the Living City together before play.
- Each player adds a landmark and rumor.
- Character questions build Bonds.
- Misses generate Adversity Tokens.
- The Living Thread is a shared stake with Aspects, Need, Boundary, and Checkpoints.
- Between sessions, players name how the city changed.

## Product Implication

The manual can be sold as a stand-alone artifact because it contains:

- core procedures
- character creation
- safety tools
- map/chronicle tools
- moves
- episode play
- campaign play
- facilitator rules
- guide-less table mode
- starter episode
- starter Static cards
- real-world rep bridge

This makes facilitation an expansion layer rather than the only access point.
