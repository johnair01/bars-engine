# Integral Development

Ken Wilber's AQAL framework applied to software — and to navigating AI-augmented creative work.

## Why Integral Theory Is the Moral Center

BARs Engine metabolizes past creative energy (transcripts, prose, recordings) into new artifacts via AI. This raises uncanny ethical questions. Integral Theory provides an ontology strong enough to navigate them:

- **Composting, not necromancy**: Past creative selves are compostable material, not sacred relics. Transmuting them serves development, not exploitation.
- **Emotional energy is fuel, not judgment**: The system converts friction and ambiguity into creative output rather than self-recrimination.
- **Dual-track integrity**: AI-augmented and non-AI paths are both first-class. The community context (Portland's reservations about AI) is respected, not overridden.

## Four Quadrants

| Quadrant | Focus | Check |
|----------|-------|-------|
| **UL (I)** | Intent, clarity | Functions readable? Names meaningful? Is the ethical footing clear? |
| **UR (It)** | Behavior | Tests pass? Performance OK? Does it degrade gracefully without AI? |
| **LL (We)** | Shared understanding | Docs clear? Patterns consistent? Would the Portland community trust this? |
| **LR (Its)** | Systems | Architecture sound? Deploys cleanly? Dual-track maintained? |

## Development Stages

- **Purple/Red**: It works (basic functionality)
- **Blue**: It's reliable (structure, rules)
- **Orange**: It's optimized (performance, UX)
- **Green**: It's empathetic (accessibility, delight, community-sensitive)
- **Yellow+**: It evolves (emergence, adaptation, creative composting)

## Code Review Checklist

- [ ] UL: Is the intent clear?
- [ ] UR: Does it work correctly?
- [ ] LL: Will others understand it?
- [ ] LR: Does it fit the system?
- [ ] Does it degrade gracefully in non-AI mode?
- [ ] Would this be legible as an in-game artifact?
