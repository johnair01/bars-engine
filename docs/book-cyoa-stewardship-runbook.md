# Book → CYOA stewardship runbook (Phase A — 1st party)

Canonical spec: [.specify/specs/book-cyoa-stewardship/spec.md](../.specify/specs/book-cyoa-stewardship/spec.md).

## Roles

- **Source steward** — owns accuracy of the adaptation vs the source work; approves attribution copy.
- **Engine steward** — ships CYOA/adventure structure and ensures provenance hooks match [campaign-ontology-alignment](../.specify/specs/campaign-ontology-alignment/spec.md) as it lands.

## Phase A workflow (internal / 1st party only)

1. **Record the source** — Title, version or edition, internal location (repo path, doc id, or Notion link). Confirm the organization has rights to adapt for the pilot.
2. **Define the adaptation unit** — Which campaign or adventure id will carry the CYOA; map major sections to passages or steps.
3. **Attribution** — Draft the player-visible block (title + credit line + optional link). Steward sign-off required before publish.
4. **Review gate** — Content review for tone and Integral/community norms; technical review for routing and flags.
5. **Publish** — Deploy through normal release process; note pilot id in `book-cyoa-stewardship` `tasks.md` when checked off.

## Phase B (third party)

Do not start until Phase A acceptance criteria in the spec are met. Then follow the deferred checklist in [.specify/specs/book-cyoa-stewardship/tasks.md](../.specify/specs/book-cyoa-stewardship/tasks.md) (licensing draft, permission workflow). This document is not legal advice.
